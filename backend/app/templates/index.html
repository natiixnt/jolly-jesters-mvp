Jak zaczac?

System sprawdza ceny produktow na Allegro na podstawie kodow EAN.

  • Dodaj kategorie w ustawieniach (np. "perfumy" z mnoznikiem 1.25 i prowizja 10%)
  • Wgraj proxy w ustawieniach
  • Przygotuj plik Excel z kolumnami: EAN, cena zakupu
  • Nowa analiza - wrzuc plik, wybierz kategorie, kliknij Start

Format pliku

Obslugiwane: .xlsx, .xls, .csv. Wymagane kolumny:

  • EAN - kod kreskowy (8-13 cyfr)
  • cena zakupu - Twoja cena nabycia
  • nazwa - opcjonalna
  • waluta - opcjonalna (domyslnie PLN)

Jak czytac wyniki?

  • Cena Allegro - najnizsza cena na Allegro
  • Marza - roznica miedzy cena Allegro a cena zakupu (minus prowizja)
  • Oplacalny - tak, jesli marza przekracza mnoznik
  • Sprzedanych - ilosc sprzedanych sztuk

Metryki i koszty

  • EAN/min - predkosc przetwarzania
  • Koszt/1000 EAN - estymacja (z CAPTCHA solving)
  • Retry rate - % zapytan ponawianych
  • Latency P50/P95 - czas odpowiedzi

Stop-loss

System automatycznie zatrzymuje analize przy problemach:

  • Error rate - zbyt wiele bledow (domyslnie > 50%)
  • Captcha rate - zbyt wiele CAPTCHA (> 80%)
  • Consecutive errors - seria bledow z rzedu (10)

Proxy

  • Wgraj plik .txt w ustawieniach (1 proxy na linie)
  • System automatycznie ocenia jakosc (health score 0-1)
  • Slabe proxy trafiaja na kwarantanne (15 min)
  • Zakladka Proxy pool pokazuje aktualny stan

Kategorie

  • Mnoznik - min marza (np. 1.25 = min 25% ROI)
  • Prowizja - % prowizji Allegro (np. 10%)
  • Kazda kategoria moze miec inne parametry

Analiza z bazy

Mozesz ponownie analizowac produkty bez uploadu pliku. System uzyje danych juz zapisanych.

API i eksport

  • Excel - pelne wyniki z metrykami
  • CSV metryk - podsumowanie runu
  • REST API - /api/v1/ (analysis, categories, settings)
  • Prometheus - /api/v1/metrics/prometheus

Rozwiazywanie problemow

  • "brak proxy" - wgraj plik w ustawieniach
  • "brak kategorii" - dodaj w ustawieniach
  • Duzo blocked - sprawdz proxy pool
  • Run stopped - stop-loss wykryl problem
  • Limit analiz - max 3 jednoczesnie
