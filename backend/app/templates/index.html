pomoc i dokumentacja

jak zaczac?

Jolly Jesters automatycznie sprawdza ceny produktow na Allegro na podstawie kodow EAN. Aby rozpoczac:

  • dodaj przynajmniej jedna kategorie w ustawieniach (np. "perfumy" z mnoznikiem 1.25 i prowizja 10%)
  • wgraj plik proxy w ustawieniach - scraper potrzebuje proxy do pracy
  • przygotuj plik Excel/CSV z kolumnami: EAN, nazwa (opcjonalnie), cena zakupu
  • przejdz do "nowa analiza", wrzuc plik, wybierz kategorie i kliknij start

format pliku

system obsluguje pliki .xlsx, .xls i .csv. wymagane kolumny:

  • EAN - kod kreskowy produktu (8-13 cyfr)
  • cena zakupu - cena po jakiej kupujesz produkt
  • nazwa - opcjonalna, do identyfikacji w wynikach
  • waluta - opcjonalna (domyslnie PLN), jesli kupujesz w EUR/USD system przeliczy automatycznie

jak czytac wyniki?

  • cena Allegro - najnizsza cena znaleziona na Allegro dla danego EAN
  • marza - roznica miedzy cena Allegro a cena zakupu (minus prowizja)
  • oplacalny - "tak" jesli marza przekracza ustawiony mnoznik
  • sprzedanych - ilosc sprzedanych sztuk w ostatnim czasie

metryki i koszty

system zbiera metryki dla kazdego runu:

  • EAN/min - przepustowosc przetwarzania
  • koszt/1000 EAN - estymowany koszt (glownie z CAPTCHA solving)
  • retry rate - procent zapytan wymagajacych ponowienia
  • captcha rate - procent zapytan z CAPTCHA
  • blocked rate - procent zablokowanych zapytan
  • latency P50/P95 - czas odpowiedzi (mediana i 95 percentyl)

metryki sa dostepne w podgladzie runu oraz do pobrania jako CSV

stop-loss (ochrona kosztowa)

system automatycznie zatrzymuje analize gdy wykryje problemy:

  • error rate - zbyt wiele bledow (domyslnie > 50%)
  • captcha rate - zbyt wiele CAPTCHA (domyslnie > 80%) - sygnal wysokiego kosztu
  • consecutive errors - seria bledow z rzedu (domyslnie 10)

progi mozna konfigurowac w ustawieniach. po zatrzymaniu zobaczysz banner z przyczynaml

proxy i siec

  • scraper wymaga proxy do pracy - wgraj plik .txt w ustawieniach
  • proxy pool automatycznie ocenia jakosc kazdego proxy (health score 0-1)
  • slabe proxy sa automatycznie wycofywane na 15 minut (kwarantanna)
  • zakladka "proxy pool" pokazuje aktualny stan wszystkich proxy

kategorie i oplacalnosc

  • mnoznik oplacalnosci - minimalny mnoznik marzy (np. 1.25 = produkt oplacalny gdy marza >= 25% ceny zakupu)
  • prowizja Allegro - procent prowizji pobranej przez Allegro (np. 10% = 0.10), odliczany od ceny sprzedazy
  • kazda kategoria moze miec inne parametry

analiza z bazy

mozesz ponownie przeanalizowac produkty bez uploadu pliku. system uzyje danych juz zapisanych w bazie. filtry: ostatnie N dni, EAN zawierajacy tekst. przydatne do re-analizy lub sprawdzenia zmian cen.

API i eksport

  • Excel (.xlsx) - pelne wyniki z metrykami scrapingu
  • CSV metryk - podsumowanie runu (koszt, przepustowosc, bledy)
  • REST API - pelne API pod /api/v1/ (analysis, categories, settings, proxies, metrics)
  • Prometheus - metryki pod /api/v1/metrics/prometheus

typowe problemy

  • "brak proxy" - wgraj plik proxy w ustawieniach
  • "brak kategorii" - dodaj kategorie w ustawieniach
  • duzo blocked/error - sprawdz jakosc proxy w zakladce proxy pool
  • run zatrzymany (stopped) - stop-loss wykryl problemy, sprawdz metryki runu
  • wolne przetwarzanie - sprawdz EAN/min w metrykach, rozważ lepsze proxy
  • limit analiz - max 3 rownoczesne analizy, poczekaj na zakonczenie
