Jak zacząć?

System sprawdza ceny produktów na Allegro na podstawie kodów EAN.

  • Dodaj kategorię w ustawieniach (np. "perfumy" z mnożnikiem 1.25 i prowizją 10%)
  • Wgraj proxy w ustawieniach
  • Przygotuj plik Excel z kolumnami: EAN, cena zakupu
  • Nowa analiza - wrzuć plik, wybierz kategorię, kliknij Start

Format pliku

Obsługiwane: .xlsx, .xls, .csv. Wymagane kolumny:

  • EAN - kod kreskowy (8-13 cyfr)
  • cena zakupu - Twoja cena nabycia
  • nazwa - opcjonalna
  • waluta - opcjonalna (domyślnie PLN)

Jak czytać wyniki?

  • Cena Allegro - najniższa cena na Allegro
  • Marża - różnica między ceną Allegro a ceną zakupu (minus prowizja)
  • Opłacalny - tak, jeśli marża przekracza mnożnik
  • Sprzedanych - ilość sprzedanych sztuk

Metryki i koszty

  • EAN/min - prędkość przetwarzania
  • Koszt/1000 EAN - estymacja (z CAPTCHA solving)
  • Retry rate - % zapytań ponawianych
  • Latency P50/P95 - czas odpowiedzi

Stop-loss

System automatycznie zatrzymuje analizę przy problemach:

  • Error rate - zbyt wiele błędów (domyślnie > 50%)
  • Captcha rate - zbyt wiele CAPTCHA (> 80%)
  • Consecutive errors - seria błędów z rzędu (10)

Proxy

  • Wgraj plik .txt w ustawieniach (1 proxy na linię)
  • System automatycznie ocenia jakość (health score 0-1)
  • Słabe proxy trafiają na kwarantannę (15 min)

Kategorie

  • Mnożnik - min marża (np. 1.25 = min 25% ROI)
  • Prowizja - % prowizji Allegro
  • Każda kategoria może mieć inne parametry

API i eksport

  • Excel - pełne wyniki z metrykami
  • CSV - podsumowanie runu
  • REST API - /api/v1/
  • Prometheus - /api/v1/metrics/prometheus

Rozwiązywanie problemów

  • "brak proxy" - wgraj plik w ustawieniach
  • "brak kategorii" - dodaj w ustawieniach
  • Dużo blocked - sprawdź proxy pool
  • Run stopped - stop-loss wykrył problem
  • Limit analiz - max 3 jednocześnie
